8. OG IMAGE GOTCHAS (for whoever edits the card next)
-------------------------------------------------------
`app/_og/card.tsx` renders in Satori, not a browser:
  - Use `logo-footer.png`, NOT `logo-hero.png`. The hero wordmark is the dark
    variant for the cream hero and is invisible on bark. This was caught by
    actually looking at the generated PNG.
  - Satori CANNOT parse WOFF2. The font must stay TTF. Montserrat-Bold.ttf ships
    as a repo asset so the build has no network dependency.
  - CSS custom properties do not resolve. Use literal hex.
  - `<img>` is required; next/image has no meaning there. The eslint rule is
    suppressed with a comment explaining why.
LinkedIn caches previews aggressively — re-check via their Post Inspector after
any change.


9. ENVIRONMENT GOTCHAS
------------------------
**git push hangs silently (cost ~40 minutes this session).**
Symptom: `git push` produces no output and times out, while `git ls-remote` and
`git fetch` work instantly. That asymmetry is the tell — the repo is public, so
reads need no auth and only the push touches the credential layer.
`GIT_TRACE=1 GIT_CURL_VERBOSE=1` shows the stall inside `git credential-manager
get`, blocking ~60s trying to raise a GUI dialog that cannot display in a
non-interactive session. `GIT_TERMINAL_PROMPT=0` does NOT help — it suppresses
terminal prompts, not GCM's GUI flow.

  FIX:  git -c credential.helper= -c credential.helper=store push origin main

The empty value is REQUIRED. `credential.helper` is multi-valued (manager,
store), so `-c credential.helper=store` alone just appends and `manager` still
runs first and still hangs.

**The older handoff's advice on this is wrong.** It says to retry with
`run_in_background: true`. Background push hangs too — it is the same blocked
process, just not in the foreground. Use the credential-helper reset above.

**Commit messages via the Bash tool: use a heredoc.** PowerShell here-string
syntax (@'...'@) is NOT valid in Bash and silently produced a commit with "@" as
the subject line this session.

**Dev server:** `npm run dev`. It may already be running on a stale port from a
previous session; Next refuses to start a second one and prints the PID to kill.
Do not assume port 3000.

**Vercel deploy-block history:** was caused by the pushing GitHub account not
being an explicit collaborator on the then-private repo, which is what Vercel
Hobby's check keys off. Fixed by making the repo PUBLIC. Not the co-author
trailer, which was the first and wrong diagnosis. `.claude/settings.json` sets
`includeCoAuthoredBy: false` regardless.
